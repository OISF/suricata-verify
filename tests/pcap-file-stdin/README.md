# Test

Read a pcap from stdin and confirm Suricata still alerts on a content match in
the packet payload. This exercises the non-seekable file handling fix for
<https://redmine.openinfosecfoundation.org/issues/8464>.

# Behavior

We expect one alert with `sid` 1 on the reassembled TCP payload. Without the
fix `InitPcapFile` calls `setvbuf` on the FILE underlying the pcap handle
after libpcap has already consumed the pcap header, glibc cannot recover on
a non-seekable fd, and the first `pcap_next_ex` returns -1, so the run
produces no alerts at all.

# Pcap

Synthetic pcap with a single TCP packet, generated by `gen-input-pcap.py`
with scapy. The TCP payload contains the deterministic content the rule
matches on.

# Command

The default invocation is overridden to pipe `input.pcap` into Suricata via
`/dev/stdin` so the read path goes through the non-seekable branch of the
fix.
