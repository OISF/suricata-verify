# bug-8232-defrag-first-frag-out-of-order

Regression test for Redmine #8232 and OISF/suricata PR #15403.

## Purpose

Exercise the RB tree neighbour search in `DefragInsertFrag` when a
fragment with `frag_offset == 0` arrives while the fragment tree is
already populated. Before the fix this seeded `RB_NFIND` with
`frag_offset - 1`, which wrapped a `uint16_t` to 65535 and forced the
search through an unintended fallback path. After the fix the key is
`frag_offset` and the search returns the correct neighbour directly.

The reassembled output is identical with or without the fix, so the
test does not fail on the pre-fix code. It serves as a permanent
regression check that future changes to the defrag neighbour search
keep reassembly correct in this ordering pattern.

## PCAP

Generated by `frag-ip-tcp-payload.py` with scapy. An IPv4 TCP packet
carrying a 40 byte payload is fragmented into eight 8 byte fragments
and written in reverse order on the wire, so the offset-0 fragment
arrives last and the search hits a non-empty tree.

## Rule

Alerts once on the reassembled TCP payload via content match.
