# Redmine #8550 defrag nested GRE fragments

Regression test for https://redmine.openinfosecfoundation.org/issues/8550.

The pcap is generated by `make-pcap.py` with Scapy. It contains one outer
IPv4/GRE datagram split into the minimum two fragments. The reassembled payload
is an IPv4/GRE fragment using the same defrag key as the outer packet.
Vulnerable builds decode the reassembled outer packet while still holding the
defrag tracker mutex, then try to lock the same tracker for the inner fragment
and hang.

The test runs Suricata under `timeout`; a fixed build completes and records one
IPv4 reassembly plus the nested fragment attempt.
