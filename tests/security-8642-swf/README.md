# SWF decompression depth wrap regression

Purpose: exercise HTTP `file_data` inspection of a compressed SWF response while
SWF decompression is enabled with a wrapping `decompress-depth` value. A
vulnerable ASAN build aborts in `FileSwfDecompression()` after the configured
`uint32_t` depth wraps during the internal FWS buffer length calculation.

The pcap is generated by `make-pcap.py` with Scapy. It contains a single HTTP
transaction returning a small zlib-compressed `CWS` body. The rule uses
`file_data`, which causes the HTTP SWF decompression path to be reached.

SHA256: `1aee2558888d337904face20909e318298440604d19b9fbcc00e1f16d8d7e626`.

The failure case is an ASAN `heap-buffer-overflow` at
`src/util-file-decompression.c:149`, where `memset()` receives a wrapped length
of `4294967295`.

PCAP generation script created by GPT 5.5 with Codex.
