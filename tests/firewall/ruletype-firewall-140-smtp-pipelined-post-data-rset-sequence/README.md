SMTP firewall regression: a second message envelope is pipelined after `RSET`
but before the previous DATA completion reply.

This pcap is generated by the email sandbox scenario
`pipelined-post-data-rset-sequence` in `smtp-oddseq.py`.

The important sequence is:

```
<--- 250-PIPELINING

---> mail from:<alice@example.com>
<--- 250 2.1.0 Ok
---> rcpt to:<bob@example.com>
<--- 250 2.1.5 Ok
---> data
<--- 354 End data with <CR><LF>.<CR><LF>

---> (message body)
---> .
---> RSET
---> mail from:<bob@example.com>
---> rcpt to:<alice@example.com>
---> data
<--- 250 2.0.0 Ok: queued as <id>
<--- 250 2.0.0 Ok
<--- 250 2.1.0 Ok
<--- 250 2.1.5 Ok
<--- 354 End data with <CR><LF>.<CR><LF>

---> (message body)
---> .
<--- 250 2.0.0 Ok: queued as <id>
```

The test asserts that the second sender and recipient belong to a second SMTP
transaction and that the firewall txbits set on that second transaction allow
the second DATA body.
