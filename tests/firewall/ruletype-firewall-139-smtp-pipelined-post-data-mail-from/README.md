SMTP firewall regression: a second MAIL FROM is sent before the previous DATA
completion reply.

This pcap is generated by the email sandbox scenario
`pipelined-post-data-mail-from` in `smtp-oddseq.py`.

The important sequence is:

```
<--- 250-PIPELINING

---> mail from:<alice@example.com>
<--- 250 2.1.0 Ok
---> rcpt to:<bob@example.com>
<--- 250 2.1.5 Ok
---> data
<--- 354 End data with <CR><LF>.<CR><LF>

---> (message body)
---> .
---> mail from:<bob@example.com>
<--- 250 2.0.0 Ok: queued as <id>
<--- 250 2.1.0 Ok

---> rcpt to:<alice@example.com>
<--- 250 2.1.5 Ok
---> data
<--- 354 End data with <CR><LF>.<CR><LF>
---> (message body)
---> .
<--- 250 2.0.0 Ok: queued as <id>
```

The test asserts that the second sender and recipient belong to a second SMTP
transaction instead of being attached to the previous DATA transaction.
