# Description

SMTP pipelining regression where a RSET and the next MAIL FROM are pipelined
in one segment right after the first message fully completed. The completed
first transaction has been freed when the RSET is parsed, so the RSET has no
owning transaction; its 250 reply must not be applied to the transaction
created by the pipelined MAIL FROM, which would complete it early and split
the second message's envelope and body across two transactions.

# PCAP

The pcap is generated by the email sandbox scenario
`pipelined-rset-after-complete` in `smtp-oddseq.py`.

The test checks that both messages are logged as exactly two transactions
with their full SMTP envelope fields and parsed email headers.
